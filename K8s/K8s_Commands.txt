Display all available k8s clusters
- kubectl config get-contexts

Select K8s cluster:
- kubectl config use-context [CLUSTER_NAME]

- Display all namespaces in K8s cluster:
- kubectl get ns OR kubectl get namespaces

Set namespace so all future k8s commands will execute within it:
- kubectl config set-context --current -n markos

Check pod specs:
- kubectl get pod [POD_NAME] -n markos -o wide
- kubectl get pod [POD_NAME] -n markos -o yaml

Scale deployment:
- kubectl scale deploy ref-loader --replicas=1
- kubectl scale deploy -l fis.cd.brm.prereq=true -n ps03 --replicas 1

Edit deployment:
- kubectl edit deploy charge-engine

Load referentials:
- k exec deploy/ref-loader -- groovysh -e 'LoaderShell.uploadTradeClearingAndBackOfficeFISProvidedData()'

Enter the container:
kubectl exec deployment/ref-loader -n ps03 -- groovysh '
	LoaderShell.uploadBackOfficeAllData();
	LoaderShell.uploadBackOfficeFISProvidedData();
        LoaderShell.uploadTradeClearingFISProvidedData();
        LoaderShell.uploadTradeClearingAndBackOfficeFISProvidedData();'

SECRETS:
- kubectl get secrets -n markos
- kubectl describe secret sec-ref-schema-installer
- kubectl get secret sec-ref-schema-installer -o yaml

Create secret for docker registry to pull images:
- kubectl create secret docker-registry registrycred --docker-server=fiscd-docker-dev.docker.fis.dev --docker-username=USERNAME --docker-password=PASSWORD --namespace markos
- kubectl patch serviceaccount default -p '{"imagePullSecrets": [{"name": "registrycred"}]}' --namespace markos

Inspect event type of logs (pod creation, pod start, image pull, etc.):
- kubectl describe pod [POD_NAME] -n namespace

Inspect application logs (application is not starting, etc.):
- kubectl logs [POD_NAME] -n namespace

Enter into container:
- kubectl exec -it [POD_NAME] -n [NAMESPACE_NAME] -- bash
